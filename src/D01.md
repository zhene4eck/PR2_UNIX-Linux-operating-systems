- Расшифровка DHCP: Dynamic Host Configuration Protocol (протокол динамической настройки узла) служит для назначения IP-адреса клиенту, протокол, который автоматически выдает ip-адрес. сетевой протокол позволяющий автоматически получать ip-адрес и другие параметры, необходимые для работы в сети TCP/IP (сетевая модель передачи данных, представленных в цифровом виде. Модель описывает способ передачи данных от источника информации к получателю.). Данный протокол работает по модели "клиент-сервер". Для автоматической конфигурации компьютер-клиент на этапе конфигурации сетевого устройства обращается к так называемому _серверу DHCP_ и получает от него нужные параметры.

- Для выдачи разрешения пользователю, созданному в Part 2 sudo необходимо добавить пользователя в группу sudo командой `sudo usermod -aG sudo username`
SUDO - Super User Do, позволяет выдавать системные права администратора для выполнеия различных операций (чтобы выполнять системные задачи, такие как установка ПО, изменение системных файлов или управление службами, без необходимости постоянного входа под учётной записью `root`, что повышает безопасность и позволяет вести аудит команд.). Это безопасность, контроль доступа, подотчетность (все команды записываются в системный журнал), временное повышение прав (нужно вводить пароль после команды + работает без пароля определенное ограниченное время).

  Команда состоит из двух частей, соединённых пайпом (|): `ps aux` и `grep sshd`.**ps** — утилита для просмотра активных процессов.​
  
  **Ключи ps**

1.  **a** — показывает процессы всех пользователей, включая системные.​

2. **u** — добавляет пользовательский формат вывода (USER, %CPU, %MEM, START и др.).​

3. **x** — включает процессы без управляющего терминала (демоны вроде sshd).
   
   Вывод команды: 

    ![alt text](task8.3.png)
    
    ![alt text](task8.3.1.png)
   
   - Перезагружаю систему командой `sudo reboot`.  
   - Устанавливаем команду `netstat` -  `sudo apt update && sudo apt install net-tools -y`. После установки проверяем версию: `netstat --version`. Пакет включает также ifconfig, route и arp.. Вывод команды `netstat -tan` 

    ![alt text](task8.4.png)

   - Объяснение ключа -tan - 
     **-t** — отображает только TCP-соединения (игнорирует UDP, UNIX).​
      **-a** — включает все соединения: listening (слушающие) и established (активные).​
     **-n** — числовой формат: IP-адреса и порты без разрешения в имена (быстрее)

   - Столбцы вывода
      **Proto** — протокол (Tcp для TCP соединений).​
      **Recv-Q** — очередь входящих пакетов (0 — нормально).
      **Send-Q** — очередь исходящих пакетов (0 — нет задержек).
       **Local Address** — локальный IP:порт (0.0.0.0:2022 — слушает все интерфейсы).
       **Foreign Address** — удалённый IP:порт (_:_ — ожидает подключений).
       **State** — статус (LISTEN — слушает входящие соединения для sshd).

   - Значение 0.0.0.0
     Адрес 0.0.0.0 означает "все доступные IP-интерфейсы" (anyaddr): sshd принимает подключения с любого сетевого адреса сервера. Это стандартно для серверов; для безопасности укажите конкретный IP в sshd_config

## Part 15. Использование планировщика заданий **CRON**

- Чтобы использовать `cron` в Ubuntu, вводим `crontab -e` для редактирования задач; в открывшемся редакторе добавим строки в формате: **`минута час день_месяца месяц день_недели команда`**, используя звездочки `*` для "каждый", и сохраняем файл, чтобы cron начал выполнение по расписанию.
  Строка `*/2 * * * * /usr/bin/uptime`
  - `*/2`: Запускать каждую вторую минуту.
  - `* * * *`: Для всех часов, дней месяца, месяцев, дней недели.
  - `/usr/bin/uptime`: Полный путь к команде `uptime` (на некоторых системах может быть `/bin/uptime`).

- Находим в системных журналах строчки (минимум две в заданном временном диапазоне) о выполнении.
